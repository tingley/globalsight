Details

 

Job ID:

The job ID assigned by the system.

Job Name:

This is the name given to the job by the initiator or automatically generated by the system.

Initiator:

The name of the user who created the job, or the project manager if the job was automatically created.

Source Locale:

The source locale of the content in this job.

Project:

Name of the project.

Example: World Services R&D

Localization Profile:

The localization profile used to create this job.

This determines the workflows and their activities for this job.

Date Created:

Date and time that the job was created.

Data Source:

The file profile used to import the content in these pages.

Pages:

The number of pages in the job. This field is part of the Job Costing feature.  Click to edit the number of pages in the job.

Priority:

The priority assigned to this job.

Workflows

 

Comments

Show the comments that have been posted for this workflow by participants.

Target Locale

The target locale of the workflow.

Word Count

Lists the number of exact, fuzzy, no match and no match repetitions  for each workflow in the job.

% Complete

The percent that the workflow is complete, calculated by activity. For example, if one of three workflow activities is complete, the "% Complete" column will show "33%".

State

The state of the workflow. Each workflow in a job can be in a different state. The states are described below:

ARCHIVED - The workflow has been archived.

BATCH_RESERVED - The workflow is waiting for all the pages, that were selected on the Import screen, to be imported. During this waiting period, the job will be in the Pending status. Once all pages have been imported, the workflow will move to the Ready status, if it is a Manual Job, or to In Progress, if it is an Automatic job.

CANCELLED - The worflow has been discarded.

DISPATCHED - The workflow is In Progress.

EXPORTED - The workflow has been exported. The job will be in the Exported status.

EXPORT_FAILED - Indicates that at least one page in the workflow failed to be exported successfully.

IMPORT_FAILED -  Indicates that at least one page in the workflow failed to be imported successfully.

LOCALIZED - All activities in the workflow have been completed. The workflow is ready for export.

PENDING - The workflow is waiting for the specified word count or timer condition to be reached. After this condition is met, the workflow will move to the Ready status, if it is a Manual job, or to the In Progress status, if it is an Automatic job.

READY_TO_BE_DISPATCHED - The workflow is waiting in the Ready status, waiting to be Dispatched.

Job Costing (Optional)

The Job Costing section of the Job Details screen is only available if you have enabled Job Costing during the installation of GlobalSight.  Internal costs typically represent those costs that you must pay to vendors or internal resources to complete a job.  Billing charges typically represent what is billed to the client requesting the localization.  

Estimated Internal Costs/Billing Charges:

The cost calculated by multiplying the Estimated Hours or the Estimated Pages by the respective rate, usually before work has begun on the job. This cost gives the Project Manager a way to save and track a cost estimate before the job is complete.

Actual Internal Costs/Billing Charges:

The cost calculated by multiplying the work completed to date by the respective rate. This is similar to a "running total" of the rate-based job cost.

Surcharges:

Non-rate-based charges that a Project Manager can add on to a job at anytime. There are two types of surcharges: Flat and Percentage. Flat surcharges are a simple amount, i.e., $30.00, that a Project Manager can add to a job. Percentage surcharges are a percent of the Actual Cost that a Project Manager can add to a job. For example, if the Actual Cost is $100.00, and I add a Percentage Surcharge of 20%, then the Final Cost will be $120.00.

Final Internal Cost/Billing Charges:

The cost calculated by adding the Actual Cost and the Surcharges. The Project Manager can override this Final Cost by clicking the Edit link and entering any Final Cost they choose. This Final Cost then becomes an Override Final Cost, meaning it has not been "calculated", rather it has been entered manually. The Project Manager can remove the override cost and revert the Final Cost to a calculated cost by checking the Remove Override checkbox on the Edit Final Cost screen.

Cost Report (Optional)

Launches a report detailing the costs for each workflow and each activity in the workflow.
