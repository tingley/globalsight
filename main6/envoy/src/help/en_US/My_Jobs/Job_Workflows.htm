Job Workflows

Summary

Job ID:

The job ID assigned by the system.

Job Name:

This is the name given to the job by the initiator or automatically generated by the system. Job name can be renamed via clicking "Edit" link(for "Ready" jobs only).

Initiator:

The name of the user who created the job, or the project manager if the job was automatically created.

Source Locale:

The source locale of the content in this job.

Project:

Name of the project.

Example: World Services R&D

Localization Profile:

The localization profile used to create this job.

This determines the workflows and their activities for this job.

Date Created:

Date and time that the job was created.

Priority:

The priority assigned to this job.

Number Of Languages:

Total target languages of the job.

Total Source Word Count:

Total word counts of source files.

PPTX Slide Total Number:

Total slides in all source pptx files (not applicable to files handled by MS Office 2007 Converter).

DOCX Page Total Number:

Total pages in all source docx files (not applicable to files handled by MS Office 2007 Converter).

Workflows

 

Target Locale

The target locale of the workflow.

Word Count

Lists the number of exact, fuzzy, no match and no match repetitions  for each workflow in the job.

% Complete

The percent that the workflow is complete, calculated by activity. For example, if one of three workflow activities is complete, the "% Complete" column will show "33%".

State

The state of the workflow. Each workflow in a job can be in a different state. The states are described below:

ARCHIVED - The workflow has been archived.

BATCH_RESERVED - The workflow is waiting for all the pages, that were selected on the Import screen, to be imported. During this waiting period, the job will be in the Pending status. Once all pages have been imported, the workflow will move to the Ready status, if it is a Manual Job, or to In Progress, if it is an Automatic job.

CANCELLED - The worflow has been discarded.

DISPATCHED - The workflow is In Progress.

EXPORTED - The workflow has been exported. The job will be in the Exported status.

EXPORT_FAILED - Indicates that at least one page in the workflow failed to be exported successfully.

IMPORT_FAILED -  Indicates that at least one page in the workflow failed to be imported successfully.

LOCALIZED - All activities in the workflow have been completed. The workflow is ready for export.

PENDING - The workflow is waiting for the specified word count or timer condition to be reached. After this condition is met, the workflow will move to the Ready status, if it is a Manual job, or to the In Progress status, if it is an Automatic job.

READY_TO_BE_DISPATCHED - The workflow is waiting in the Ready status, waiting to be Dispatched.

Current Activity

Shows the current activity the workflow is in

Estimated Translate Completion Date

Shows the estimated completion date of the last translate type activity in the default path of the workflow.

 

Estimated Workflow Completion Date

Shows the estimated completion date of the workflow (see Workflow Details).

Uploading

Indicate if current activity are uploading and importing offline files back to GlobalSight.

Yes: The uploading is processing. Export target files is not allowed.

No: No uploading is processing. Export target files is allowed.

Buttons

 

Re-Assign

Re-assign activities for qualified users (see Re-Assign User).

Discard

Discards the selected workflows.  All translations done up to this point are lost.

Update Leverage

Update leverage from in-progress jobs TM or/and re-leverage reference TMs which are specified by TM profile.

Leverage MT

Leverage MT for selected workflows. This button works when MT profile has been attached to at least one workflow and all selected workflows are ready for dispatch.

Update Word Counts

After update leverage or leverage MT, user can select to update word counts for certain workflows according to newly retrieved matches.

Detailed Word Counts...

View word counts (see Detailed Word Counts)

Translated Text...

Display the percentage of translated segments for the selected target locale(s).

Archive

Moves the selected workflow to the Archived state.

Details

Displays a read-only graphical view of the workflow and a history of all the Activities in the workflow.  Also allows the user to enter a projected completion date for the workflow, and to view any secondary target files for the workflow.

Export...

Performs an interim export of the files in the selected workflow.

Add...

Add a workflow to this job (see Add Workflow).

Edit...

Opens the workflow in the graphical editor for editing.  (See Workflow Editor).

Dispatch

Dispatches the selected workflow. This will start the workflow and move the workflow to In Progress.

Skip Activities

Force the workflows to the activity you selected. (see Skip Activities).

Download...

Select Files to Download

Export & Download...

Exports and Download the files in the selected workflows.

Previous

Returns to the previous jobs listing screen.

Download QA Reports

Batch download QA Checks reports for selected workflows. This button is only available when current project has enabled Allow Manual Running of QA Checks feature. If current activities of selected workflows are QA checks enabled activities and/or have run QA checks manually, then a new QA Checks report will be generated for each selected workflow, otherwise it will download the latest existing QA checks reports.
