Create/Modify TM Profile

Summary:

The TM profile determines translation memory options for any localization profile that uses it.  The options include where and when translations are stored, as well as leveraging options.

Basic Info

 

Name

For display in GlobalSight. The name may only contain letters, numbers, the underscore "_" or the hyphen "-".

Description

Optional description for this TM profile.

SRX Rule set

Select a Segmentation Rule to relate with this TM Profile.

TM Population Options

 

Save to TM

The TM selected from the dropdown is where source and target segments are stored.

Save unlocalized segments to TM

Checked: segments that have not been changed (unlocalized) are written to the TM.  These segments will leverage in the future and reduce wordcount.

Unchecked: segments that have not been changed (unlocalized) are not written to the TM.  These segments will not leverage in the future, and will result in a higher wordcount.

Save localized segments to TM

Checked: segments that have been changed (localized) are written to the TM. These segments will leverage in the future and reduce wordcount.

Unchecked: segments that have been changed (localized) are not written to the TM. These segments will not leverage in the future, and will result in a higher wordcount.

Save MTed segments to TM

Checked: segments that translated by MT are written to the TM.  These segments will leverage in the future and reduce wordcount.

Unchecked: segments that translated by MT are not written to the TM.  These segments will not leverage in the future, and will result in a higher wordcount.

Save Wholly Internal Text Segments to TM

Checked: segments that wholly internal text are written to the TM.  These segments will leverage in the future and reduce wordcount.

Unchecked:segments that wholly internal text are not written to the TM.  These segments will not leverage in the future, and will result in a higher wordcount.

Save exact match segments to TM

Checked: segments that exact match are written to the TM.  These segments will leverage in the future and reduce wordcount.

Unchecked: segments that exact match are not written to the TM.  These segments will not leverage in the future, and will result in a higher wordcount.

Save approved segments to TM

Checked: approved segments and multiple exact match segments with SID are written to TM. These segments will leverage in the future and reduce wordcount.

Unchecked: approved segments are not written to TM. Multiple exact match segments with/without SID are not written to TM unless the checkbox "Save unlocalized segments to TM" is checked. These segments will not leverage in the future, and will result in a higher wordcount.

Save unlocalized segments to Page TM

Checked: segments that have not been changed (unlocalized) are written to the Page TM.  These segments will leverage on future re-imports and reduce wordcount.

Unchecked: segments that have not been changed (unlocalized) are not written to the Page TM.  These segments will not leverage on future re-imports, and will result in a higher wordcount.

General Leverage Options

 

List of Exclude Item types:

List any item types that should be excluded from translation.

Leverage Localizables

Checked: localizables are leveraged.

Unchecked: localizables are not leveraged.

Leverage Exact Matches Only

Checked: the system searches for exact matches only, not for fuzzy matches.  Although this may improve import performance, no fuzzy matches are available

to assist the localization process.

Unchecked: the system searches for both fuzzy and exact matches during the leverage process.

Leverage in-context matches

Apply SID ICE Promotion Only:Only SID matched 100% matches can be promoted to in context match.

Apply SID/Hash ICE Promotion:SID and hash matched 100% matches can be promoted to in context match.

Apply SID/Hash/Bracketed ICE Promotion:SID, hash and bracketed matched 100% matches can be promoted to in context match.

Dynamic Leverage Options

 

Leverage Approved translations from selected Reference TM(s)

Checked: When a segment is opened the Reference TM(s) are checked for new translations.  If any exist, the translator is presented with these matches.

Unchecked: No TM(s) are referenced when the translator opens a segment.

Leverage In-progress translations from the Job

Checked: When a segment is opened all in-progress translations for the same job are checked to see if any suitable matches exist.  In-progress translations would consist almost entirely of the translator's own work.

Unchecked: In-progress translations are not referenced when the translator opens a segment.

and from Jobs that write to the Storage TM

Checked: When a segment is opened all in-progress translations that will be written to the same Storage TM are checked to see if any suitable matches exist.

Unchecked: In-progress translations from the storage TM are not referenced when the translator opens a segment.

and from Jobs that write to selected Reference TM(s)

Checked: When a segment is opened all in-progress translations that will be written to any of the reference TMs are checked to see if any suitable matches exist.

Unchecked: In-progress translations from reference TMs are not referenced when the translator opens a segment.

Stop search after hitting 100% match

Checked: Stop hierarchical TMs searching after hitting 100% match

Unchecked: Continue searching TMs after hitting 100% match

Leverage Options for Initial Import

 

Leverage From TM:

Select one or more TMs to use for leveraging.  Use ctrl+click to select additional TMs after the first one.

Type-sensitive leveraging:

Checked: The type (e.g. URL, Text, Alt) of a new segment is compared to segments in the TM during leveraging.  If two segments differ by type, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: The type of the new segment is not compared during leveraging.

Case-senstive leveraging:

Checked: The case of a new segment is compared to segments in the TM during leveraging.  If two segments differ in case, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: The case of the segment is not compared during leveraging.

Whitespace-sensitive leveraging:

Checked: Whitespace in a new segment is compared to segments in the TM during leveraging.  If two segments differ in whitespace, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: Whitespace in the segment is not compared during leveraging.

Code-sensitive Leveraging:

Checked: Code, such as formatting, in a new segment is compared to segments in the TM during leveraging.  If two segments differ in code, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: Code in the segment is not compared during leveraging.

Reference TM:

Checked: If segment refer the selected TM, the fuzzy match percentage is reuced by the amount in the Penalty box.

Unchecked: The segment is not consider during leveraging.

Old TUV Match Leveraging:

Checked: If segment create date, modify date or last used date more than the number of days, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: The date of the segment is not compared during leveraging.

Multilingual Leveraging:

Checked: Both source and target segments in the appropriate locale are checked during leveraging.  For example, if a page was translated from English US, to French FR, and German DE, the French FR page could later be imported and leveraged into German DE.

Unchecked: Only segments which have been imported as source are checked during the leveraging process.

Auto Repair Placeholders:

Checked: Automatic repair TM entries imported from WorldServer

Unchecked: Not repair TM entries imported from WorldServer

Get Unique from Multiple Exact Matches:

Checked: Only one exact match will be returned from TM. As to which one will be returned, sort by reference TM order, latest/oldest, then return the first. All fuzzy matches will be returned if have.

Unchecked: All matches will be returned as usual.

Multiple Exact Matches:

In cases where there are more than one unique translation for an exact match:

Latest The most recent translation is selected as the exact match.

Oldest The oldest translation is selected as the exact match.

Demoted Multiple exact matches are displayed as fuzzy matches.  The match percentage is reduced by the amount in the Penalty box.

Fuzzy Match Threshold:

Matches below this threshold are not displayed.

Number Of Matches:

This is the maximum number of matches that will be displayed in the segment editor.

Leverage Options for Reimport

 

Choose Latest Match

Checked: The Page TM is not used as the initial leveraging source for re-imported pages.  The TMs specified in "Leverage From TM" above are used.  For any multiple exact matches, the latest match is used.

Unchecked: The Page TM is used as the initial leveraging source for re-imported pages.  If no exact match is found for a segment, the TMs specified in "Leverage From TM" are used.

Type-sensitive leveraging:

Checked: the type of a new segment is compared to segments in the Page TM during leveraging.  If two segments differ by type, the fuzzy match percentage is reduced by the amount in the Penalty box.

Unchecked: the type of the new segment is not compared when leveraging from the Page TM.

No Multiple Exact Matches:

In cases where there are more than one unique translation for an exact match:

Checked: Exact matches are displayed as fuzzy matches.  The match percentage is reduced by the amount in the Penalty box.

Unchecked: One of the is exact matches is arbitrarily selected as the exact match.

TU Attributes Match Prioritising Rules

Add/edit the prioritising rules for TU Attributes, add permission "Setup -> Translation Memory -> Enable TM Attributes" to view it.

Disregard if TU attributes not matched:

When TU attributes do not match the prioritising rules, if this is checked, the TM match will be ignored and not saved into leverage match. If TU attributes are matched, this will not happen.

Penalize if TU attributes not matched:

When TU attributes do not match the prioritising rules, if this is checked, penalize the TM match before save into leverage match. The penalty will be between 1 and 100. If TU attributes are matched, this will not happen.

Buttons

 

Cancel

Cancels TM profile creation/edit.

Save

Saves the TM new/modified profile.
